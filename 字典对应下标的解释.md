这个问题非常关键：**字典为什么能“输入一个值，就拿到它对应的下标”**？
答案是：**不是“字典天生知道下标”，而是“我们自己把下标存进去的”**。

---

## 1）先把“字典”理解成一个小本本（键→值）

字典（dict）就是一种“映射关系”：

> **键（key） → 值（value）**

比如：

```python
seen[7] = 1
```

这句话的意思是：

* 键 key 是 `7`
* 值 value 是 `1`

相当于在小本本上写了一行：

> 7 对应 1

之后再问它：

```python
seen[7]
```

它就会把之前写进去的 `1` 给。

所以“输入 7 输出 1”不是魔法，是之前**亲手记录**的。

---

## 2）问的“怎么知道会输出下标”——因为我们存的就是下标

在代码里我们写的是：

```python
seen[x] = i
```

这里：

* `x` 是“数组里的某个数值”
* `i` 是“这个数值所在的位置（下标）”

所以我们把“数值”当做 key，把“下标”当做 value 存进去。

因此以后当我们写：

```python
seen[need]
```

拿到的自然就是“need 这个数之前出现时的下标”。

---

## 3）用一个超小例子演示（看完就懂）

假设：

```python
nums = [3, 8, 2]
```

循环时会发生：

### 第1次：i=0, x=3

执行：

```python
seen[3] = 0
```

现在字典是：

```python
{3: 0}
```

### 第2次：i=1, x=8

执行：

```python
seen[8] = 1
```

现在字典是：

```python
{3: 0, 8: 1}
```

这时候问：

```python
seen[3]
```

就会得到 `0`，因为之前存的是 `seen[3] = 0`。

问：

```python
seen[8]
```

就会得到 `1`。

---

## 4）为什么字典能这么快找到对应的下标？

这是字典的“天赋”：它内部用一种结构（现在不用深入）让能很快查找。

只要记住两点就够了：

1. **存什么，它就记什么**
2. **用同一个 key 取值，就能取回存的 value**

---

## 5）一句话总结问的点

> 因为我们写了 `seen[x] = i`，所以字典里保存的是“数值 x → 它的下标 i”。
> 后面用 `seen[need]` 就能取回 “need 对应的下标”。

---

如果愿意，我可以继续把这一句也“逐符号”拆给看：

```python
if need in seen:
    return [seen[need], i]
```

并解释：为什么先 `in` 再 `seen[need]`，不然会报错。
